`flutter_file_hash` is a work-in-progress Flutter FFI package for hashing files
and bytes through a Zig-powered native core.

The repository is being bootstrapped. The public Dart API, native packaging, and
release workflow are not stable yet.

## Planned Scope

- One shared Zig core via `zig-files-hash`
- Dart FFI bindings
- Stream-only hashing through the Zig C ABI hasher API
- `fileHash` and `stringHash`
- Lowercase hex digest output
- Android, iOS, macOS, Linux, and Windows support
- Algorithms exposed by the `zig-files-hash` C ABI

Web is not planned for the first implementation.

## Status

Not ready for production use.

Current focus:

- adapt the Flutter `package_ffi` scaffold
- wire Dart FFI to the Zig streaming hasher API
- package native artifacts for all target platforms
- add cross-platform test vectors
- document the final Dart API

## Project Structure

- `lib/`: Dart API and FFI bindings
- `src/`: native scaffold files generated by Flutter, to be replaced/adapted
- `hook/build.dart`: native assets build hook
- `third_party/zig-files-hash`: pinned Zig hashing core
- `example/`: Flutter example app generated by the package scaffold

## Core

The hashing engine is pinned as a Git submodule:

```text
third_party/zig-files-hash
```

Initialize it after cloning:

```bash
git submodule update --init --recursive
```

## Bindings

Generated bindings currently come from the Flutter FFI template. They will be
replaced with bindings for `zig_files_hash_c_api.h`.

Regenerate bindings after the FFI configuration is finalized:

```bash
dart run ffigen --config ffigen.yaml
```

## Native Assets

The package uses Flutter's native assets build hook:

```text
hook/build.dart
```

The initial scaffold builds a small C example library. The planned
implementation will instead bundle or select Zig-built native artifacts.
